# FurColor Studio 操作手册

## 0. 开机前检查

- 备份原片；FurColor 永远不应覆盖原片目录。
- 输出目录与原片目录必须分开。
- 使用本机 `127.0.0.1:8899`，不要把本地完整版暴露到公网。
- 确认水印 PNG、参考样片和客户授权范围。

## 1. 创建项目

点击“新建项目”，依次选择：

- 原片目录：JPG/JPEG/ARW，同名文件优先使用 JPG 预览；
- 手修参考目录：用于学习本场灯光和你的后期风格，可留空；
- 分析目录：模型报告与复核中间结果；
- 输出目录：AI 草稿；
- 水印 PNG：可选，每个项目独立选择；
- 参考样片清单：可选 JSON；
- 正选或反选模式。

正选模式只有“保留”照片进入后续流程；反选模式默认进入，只有“废片”被排除。该规则同时应用于分析、眼睛标注、渲染和最终交付。

## 2. 选片

在选片台逐张标记“保留”“废片”或“默认”。缩略图由本地缓存生成，不会上传。商业活动建议先剔除失焦、闭眼、严重遮挡、重复动作与无授权画面，以节省处理和存储成本。

## 3. 分析与隐私复核

先运行“隐私与参考分析”，再打开人脸复核。真人脸检测只作为候选生成器；必须人工确认：

- 背景人物正脸与侧脸；
- 镜面、手机屏幕、玻璃中的人脸反射；
- 兽装耳朵是否被误判；
- 胸牌、二维码、群号等可识别信息。

人工反馈只保存在项目私有运行目录，不进入仓库或云端 demo。

## 4. 眼睛蒙版

每张照片单独标注兽装眼睛的多边形/椭圆轮廓。不要跨照片复用固定坐标：焦段、距离、姿态、透视与眼睛造型都会变化。增强建议保持克制，并检查蒙版羽化和高光是否自然。

## 5. V3.3 渲染与抽检

渲染后至少检查：

- 中性白区域是否真的中性，舞台有色灯是否被过度校正；
- 白毛高光是否溢出、黑毛是否仍有纹理；
- 头部与主体是否因局部测光欠曝；
- 同场景相邻照片的一致性；
- 眼睛增强是否越界；
- 水印位置、大小、透明度和裁切。

建议对同一灯光场景抽检首、中、尾照片；遇到灯光切换应增加抽检密度。

## 6. 交付

完成界面四项人工确认后，后端才允许生成交付包。交付目录包含：

- 最终 JPG；
- `manifest.json`：项目名、时间、数量、选片模式、质检声明；
- 每个文件的 SHA-256；
- 可选 ZIP。

交付前仍应使用系统图片浏览器做最后一遍全量快速检查。
