# FurColor Studio 4.0 模型卡

FurColor 4.0 是摄影师辅助流水线，不是单一端到端模型，也不是无人值守的发布审核器。系统把多个相互独立的证据组合起来，并保留人工复核门槛。

## 组成

### YuNet 真人脸候选

OpenCV YuNet 负责生成候选框和五点几何。摄影师可把候选标为 `human` 或 `fursuit`；两类各至少 2 个样本后，项目内逻辑回归记忆模型开始校准候选概率。训练准确率不是验证准确率。

### Fursee 主体智能层（可选）

Fursee 的 YOLO26l 权重检测 `furry` 与 `face` 两类；ArcFace 优化的 DINOv3 ViT-L/16 将兽头裁剪转换为 512 维归一化向量，DBSCAN 只在当前项目内形成 `C001` 等匿名分组。

FurColor 对它做了以下限制：

- Fursee 单独发现的人脸只进入人工复核，绝不直接自动打码；
- 只有高置信兽头框、低 YuNet 几何、低记忆概率且没有 Fursee 人脸证据同时满足时，才把 YuNet 候选抑制为疑似兽装误检；
- 若 Fursee 人脸与低几何 YuNet 候选重合，候选恢复为人工复核，避免被几何规则静默排除；
- 特征向量写入项目私有 `subject_embeddings.npz`，不跨项目检索，不映射到真人身份；
- 单个主体或不稳定样本可保留为噪声，不强行聚类。

### 白平衡与曝光

白平衡优先寻找不过曝、低色度的中性区域；找不到可靠区域时保持原结果，不强制猜测。曝光先分析全图波形与白/黑主导场景，再基于兽头框比较主体中位亮度与场景中位亮度，进行最多约 ±0.38 EV 的羽化局部修正。白毛接近溢出时优先压制高光，黑毛只有明显低于合理场景差值时才提亮。

## 适用范围

- 兽装活动 JPG/JPEG/ARW 批量初筛与草稿；
- 同一活动项目内的匿名连拍分组；
- 真人脸候选复核、兽耳误检减少；
- 参考样片驱动的基础影调、主体曝光、眼睛增强和水印。

## 不适用范围

- 实名身份识别、跨活动跟踪或人物画像；
- 将分组编号当作角色或兽主身份；
- 未经人工检查直接公开或商业交付；
- 判断拍摄授权、肖像权或活动合规性；
- 医疗、执法、门禁等高风险生物识别用途。

## 已验证环境

2026-07-22 在 Windows、Python 3.12、PyTorch 2.7.1 + CUDA 12.8、Ultralytics 8.4.60、Transformers 4.56.0、RTX 5060 Laptop GPU 上完成本地测试：YOLO 检测、严格权重迁移、DINOv3 FP32 前向、512 维归一化特征和 3 张样例的端到端任务均通过；同一裁剪与随包参考特征的余弦相似度为 `0.99999976`。该结果只证明软件兼容性，不代表真实活动准确率。

## 必须人工检查

真人脸与反射、胸牌和二维码、兽耳误打码、匿名分组错误、眼睛蒙版边缘、白毛高光、黑毛细节、白平衡、水印与最终裁切。任何模型输出都不能证明照片可以安全发布。
