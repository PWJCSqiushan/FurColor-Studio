# Privacy Policy for FurColor Studio

## 默认原则

FurColor Studio 采用 local-first 设计。真实照片、文件路径、人脸候选、人工反馈、眼睛标注、水印、模型权重、项目数据库和主体派生数据默认只保存在操作者本机。

## 明确不会上传的内容

- 原片、裁剪预览、草稿和交付成片；
- 客户姓名、联系方式、活动名单和授权书；
- 人脸框、误检反馈、兽装眼睛标注；
- Fursee 兽头裁剪、匿名分组和 512 维特征向量；
- 本机绝对路径、水印源文件；
- API 密钥、SSH 私钥、服务器凭据；
- 训练集、记忆库与模型权重。

## Fursee 主体数据

Fursee 只在当前项目内生成匿名 `C001` 等分组。`subject_embeddings.npz` 位于被 Git 忽略的项目运行目录，不跨项目检索，不与真人姓名、账号或联系方式关联。删除项目私有运行目录即可删除这些派生数据。FurColor 不把兽装相似性功能用于真人身份识别。

## 人脸处理边界

Fursee 单独发现的人脸只进入人工复核，不直接触发自动模糊。YuNet 候选只有在多项证据达到安全门槛时才允许自动隐私预处理。模型结果不能证明照片安全、获得授权或符合活动规则，商业交付前必须人工检查背景、反射、屏幕与印刷人脸。

## 云端演示版

云端演示版不提供文件上传、路径读取、照片处理、主体分析、交付生成或管理接口。容器不挂载宿主照片和模型目录，并以非 root 用户和只读根文件系统运行。

## 操作者责任

操作者必须确认拍摄和处理权限，遵守活动方与圈内隐私规范，并在公开发布或商业交付前人工检查真人脸、反射、胸牌、二维码、地理位置等识别信息。
